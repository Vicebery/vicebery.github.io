基本类型

数据类型

Java中，一共有8种基本类型，其中有4种整型、2种浮点类型、1种用于表示Unicode编码的字符单元的字符类型char和一种用于表示真值的boolean。

整型：

int 4字节 -21474836482147483647（大约21亿）
short 2字节 -3276832767
long 8字节 -92233720368547758089223372036854775807（大约922亿亿）
byte 1字节 -128127

浮点类型：

float 4字节 大约+-3.4E+38F(有效位数为6~7位)
double 8字节 大约+-1.79E+308(有效位数为15位)

用于表示溢出和出错情况的三个特殊的浮点值：

  • 正无穷大
  • 负无穷大
  • NaN(不是一个数字)

char类型

2个字节，16位，表示单个字符。

char 使用 UTF-16 编码。char 不能存储所有汉字，只能存储在 UTF-16 编码下占两字节的汉字，

boolean类型

只有两个值：false和true，并且不可被0或者1（非0）代替。

变量与常量

变量申请后必须初始化
在Java中，必须使用final定义常量

String类

Java中没有内置的字符串类型，而在标准Java类库中提供了一个预定义类String。每个用双引号括起来的字符串都是String类的一个实例。String类中没有提供用于修改字符串的方法。故String类对象称为不可变字符串。

空串""是长度为0的字符串，是一个Java对象，有自己的串长度（0）和内容（空）

null表示没有任何对象与该变量关联，没有分配内存。

String、StringBuilder和StringBuffer
执行速度：StringBuilder>StringBuffer>String
StringBuilder 和 StringBuffer均为字符串变量，两者的对象是变量，是可以更改的。
StringBuilder线程不安全，适用于单线程下在字符缓冲区进行大量操作的情况
StringBuffer线程安全，适用于多线程下在字符缓冲区进行大量操作的情况

多重选择：switch

case标签可以是：

  • 类型为char,byte,short,int的常量表达式
  • 枚举常量
  • 字符串字面量（从java7开始）

对象与类

对象与对象变量之间存在一个重要区别，未初始化的对象变量本质上不是一个对象，但它可以引用相应类型的对象。在Java中，任何对象变量的值都是对存储在另外一个地方的一个对象的引用。new操作符的返回值也是一个引用。所有的Java对象都存储在堆中

更改器方法与访问器方法

只访问对象而不修改对象的方法有时称为访问器方法。
调用完方法后对象的状态会改变，这种方法称为更改器方法。

预定义类

Math类
Date类
LocalDate类

使用这些预定义类能更灵活地处理这个业务。

隐式参数与显式参数

方法用于操作对象以及存取它们的实例域。
出现在方法名前的类对象称为隐式参数，方法名后括号中的参数称为显式参数。每个方法中，关键字this表示隐式函数。

注意不要编写返回引用可变对象的访问器方法。

一个类的方法可以访问该类的任何一个对象的私有域。

final实例域

将实例域定义为final，构建对象时必须初始化这样的域，并且在后面的操作中，不能够再对它进行修改。

静态域与静态方法

静态域也称类域，一个类即使没有一个对象，静态域也存在。它属于类，而不属于任何独立的对象。

静态常量

例：定义一个π的常量

静态方法

静态方法是一种不能向对象实施操作的方法。

例如：Math类的pow方法就是一个静态方法。

工厂方法

静态工厂方法可以用来构造对象

方法参数

Java采用按值传递，方法得到的是所有参数值的一个拷贝，故，方法不能修改传递给它的任何参数变量的内容。

方法参数有两种类型：

  • 基本数据类型
  • 对象引用

方法无法修改一个基本数据类型的参数，但可以修改对象引用，类似C中的指针传递。

Java的参数传递，不管是基本数据类型还是引用类型的参数，都是按值传递，没有按引用传递！
当一个实例对象作为参数被传递到方法中时，参数的值就是该对象的引用的一个副本。指向同一个对象，对象的内容可以在被调用的方法内改变，但对象的引用(不是引用的副本) 是永远不会改变的。

Java值传递和“引用传递”

构造器

如果构造器中没有显式地给域赋初值，那么就会被自动地赋为默认值：数值为0、布尔值为false、对象引用为null。

如果没有在构造器中对某些域进行初始化，就会被赋值为默认值。

无参数的构造器:当且仅当一个类没有编写构造器时，系统会为类提供一个无参数的构造器，该无参构造器为所有实例域设置为默认值。

this调用另一个构造器，当构造器第一个语句形如this（…），这个构造器将调用同一个类中的另一个构造器。

包作用域

标记为public的部分可以被任意的类使用；标记为private的部分只能被定义它们的类使用，如果没有指定public或private，这个部分（类、方法或变量）可以被同一个包中的所有方法访问。

继承extends

关键词extends表明正在构造的新类派生于一个已存在的类。已存在的类称为超类、基类或父类；新类称为子类、派生类或孩子类。

阻止继承：final类和方法

不允许扩展的类被称为final类。定义类的时候使用了final修饰符就表明这个类是final类。

类中的特定方法也可以被声明为final，这样子类就不能覆盖这个方法（final类中的所有方法自动成为final方法,但不包括其中的域）。

域声明为final后，构造对象之后就不能改变它们的值了。

覆盖方法

子类不能直接访问父类的私有方法，须通过公有接口访问。子类用新方法覆盖超类中的方法时，用super调用父类中的这个方法。

在覆盖一个方法的时候，子类方法不能低于超类方法的可见性。特别是，如果超类方法是public，子类方法一定要声明为public。

子类构造器

用super（参数）调用父类的构造器，super语句必须是子类构造器的第一条语句。

super的用途：

  • 调用超类的方法
  • 调用超类的构造器

this的用途：

  • 引用隐式参数
  • 调用该类其他的构造器

多态

一个对象变量可以指示多种实际类型的现象被称为多态。

抽象类abstract

抽象类中不仅可定义抽象方法，还可定义具体数据和具体方法，建议尽量将通用的域和方法放在超类（不管是否是抽象类）中。

抽象类不能被实例化，但可以定义一个抽象类的对象变量，且它只能引用非抽象子类的对象。

抽象方法充当着占位的角色，它们的具体实现在子类中。一旦类中声明抽象方法，那么该类必须为抽象类。

受保护的访问protected

有些时候人们希望超类中的某些方法允许被子类访问，或允许子类的方法访问超类中的某个域，需要将这些方法或域声明为protected。

Java用于控制可见性的4个访问修饰符：

  • 仅对本类可见–private
  • 对所有类可见–public
  • 对本包和所有子类可见–protected
  • 对本包可见–默认（不需要修饰符）

在同一包内，protected和默认的作用相同。当子类与超类不在同一个包时二者显现区别。

Object：所有类的超类

Object类是Java中所有类的始祖，Java中的每个类都由它扩展而来。

equals方法

Object类中的equals方法用于检测一个对象是否等于另一个对象。这个方法将判断两个对象是否具有相同的引用。

五种特性：

  • 自反性
  • 对称性
  • 传递性
  • 一致性
  • 对于任意非空引用x，x.equals(null)应该返回false

hashcode方法

散列码是由对象导出的一个整数值，hashCode方法定义在Object类中，故每个对象都有一个默认的散列码，其值为对象的存储地址。

字符串的散列码是由内容导出的，因此String new出两个相同字符串其对象拥有相同的散列码，StringBuffer new出两个相同字符串却拥有不同散列码，是因为在StringBuffer类中没有定义hashCode方法，它的散列码是由Object类的默认hashCode方法导出的对象存储地址。

泛型数组列表

ArrayList类，使用起来有点像数组，但在添加或删除元素时，具有自动调节数组容量的功能，而不需要为此编写任何代码。

ArrayList是一个采用类型参数的泛型类

List接口有三个实现类，分别是ArrayList、Vector、LinkedList。ArrayList和Vector允许对元素进行随机访问，前者不支持线程安全，后者支持线程安全，如果不考虑线程的安全因素，一般用ArrayList效率比较高。LinkedList是用链表结构存储数据的，适合数据的动态插入和删除，随机访问和遍历数组比较慢。

对象包装器和自动装箱

所有的基本类型都有一个与之对应的类（Integer->int），这些类称为包装器。Integer、Long、Float、Double、Short、Byte、Character、Void、Boolean（前6个派生于公共的超类Number）。

枚举类

接口

接口是对类的一组需求描述，这些类要遵从接口描述的统一格式进行定义。

接口中的所有方法自动地属于public，因此在接口中声明方法时，不必提供关键字public。但在实现接口时，必须把方法声明为public。

为了让类实现一个接口，通常需要以下步骤：
1） 将类声明为实现给定的接口
2） 对接口中的所有方法进行定义

接口的作用：

  • 提供多重继承的大多数好处，同时避免多重继承的复杂性和低效性

内部类

内部类是定义在另一个类中的类。内部类方法可以访问该类定义所在的作用域中的数据，包括私有的数据。内部类可以对同一个包中的其他类隐藏起来。当想要定义一个回调函数且不想编写大量代码时，使用匿名内部类比较便捷。
