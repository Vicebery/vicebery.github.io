忽略已被track的文件

有时我们会遇到这种问题，在新建项目时没有编辑.gitignore文件，导致许多无用的或者私密的文件被track，然后被提交到远程库，等我们意识到这个问题时，我们创建了.gitignore文件，并将相应的不需要提交到远程库的文件（主要是编译文件和私密文件）按照规则写到.gitignore文件中，然后提交，但却发现，.gitignore文件没有生效。已被track的文件仍然会被commit，如何解决？

该问题在Stack Overflow上的解决方案

在项目根目录下执行以下三条语句：

git rm --cached <file>这条语句意思是删除已被track文件的track标记（并不是真的删除文件）

git rm -r --cached .这句意为删除当前目录下的所有已被track文件的track标记。-r 意为递归执行，即对当前文件下的所有文件以及文件夹操作。

git add .为所有未被忽略的文件添加track标记，或者说是将本地库中需要被提交的文件添加到Git Index。在执行这行命令前，确保已经编辑好了.gitignore文件，这条命令会按照.gitignore文件设定的规则来track文件。

git commit -m "update .gitignore and it is now working"提交此次操作到远程库。
