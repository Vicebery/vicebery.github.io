fabric环境搭建以及实例运行

发表于 2018-08-03 | 分类于 fabric

0. 前言

以下操作是在ubuntu 18.04上进行，网络使用SS代理，如不使用网络代理，操作过程中可能会出现各种问题。

1. 先行步骤

具体请查看官方文档介绍，这里主要介绍两点：

1.1 安装go以及环境变量配置

建议直接命令安装：

sudo apt-get install golang

安装好后设置环境变量，在主目录下新建一个文件夹“go”，作为你的工作空间。

打开终端，编辑记录环境变量的文件：

sudo vim /etc/profile

在文件最后添加：

在系统下用命令安装不需要添加GOROOT，因为默认安装位置是/usr/lib，此路径已经被写入环境变量了。

官方文档写的是编辑~/.bashrc这个文件，关于这个问题，网上也有很多版本，这篇博客有相关介绍。

安装好后，检验一下：

sudo go env

若显示go的相关环境，则证明安装成功。

1.2 docker以及docker-compose的安装

命令直接安装一般不会成功，这里参考清华大学镜像源安装。

按照以上网址的教程，依然出现问题：
在导入信任docker的GPG公钥时，由于国内网络问题，docker官网不能正常访问，出现错误：

curl: (7) Failed to receive SOCKS4 connect request ack.
gpg: 找不到有效的 OpenPGP 数据。

于是在终端下设置临时代理：

export https_proxy=socks5://127.0.0.1:1080

显示“ok”，即导入成功。

2. fabric部署

1. 创建工作目录

在你的go的工作空间中创建文件夹

mkdir -p ~/go/src/github.com/hyperledger/

2. 下载fabric源码

进入到刚刚创建的hyperledger文件夹，git clone源码

cd ~/go/src/github.com/hyperledger
git clone https://github.com/hyperledger/fabric.git

如果git clone很慢，可以设置git的网络代理：

git config –global http.proxy ‘socks5://127.0.0.1:1080’
git config –global https.proxy ‘socks5://127.0.0.1:1080’

3. 下载镜像文件

上一步会在hyperledger文件夹下创建一个新文件夹fabric，找到fabric文件夹里的scripts文件夹里的bootstrap.sh脚本，这个脚本就是下载相关镜像的，运行这个脚本：

./bootstrap.sh

如果由于网络问题下载不成功，建议先在终端设置代理：

export http_proxy=”socks5://127.0.0.1:1080”
export https_proxy=”socks5://127.0.0.1:1080”

由于镜像文件有点多，可能需要花费一些时间。下载完成后会罗列出下载的docker镜像，有好几个竟然显示几个G，很疑惑这些文件在哪儿？去脚本文件查阅也没找出来

在scripts文件夹下有个fabric-samples文件夹，在上一步完成后会出现一个bin文件夹，将这个文件夹复制到fabric/examples下

4. 运行实例

接下来运行实例：e2e_cli，在该文件夹下打开终端：

./network_setup.sh up

会输出一大堆东西，最后看到字符打印出“END-E2E”。

关闭实例，按Ctrl+c重回终端输入：

./network_setup.sh down

ubuntu安装的相关问题

发表于 2018-08-02 | 分类于 ubuntu

以下介绍用U盘安装ubuntu与windows共存的一些问题：

1.确认安装的位置：

想要ubuntu和windows共存，就要为ubuntu单独预留一份空间，如果你本身的系统是windows，可以在磁盘管理下，选定某个盘符，点击右键压缩卷，压缩出来的空间作为ubuntu安装的存储空间，大小按个人需求来定，像我学习fabric等区块连平台，预留了50G。压缩卷后看到有一块变成绿色或者灰色，即可用或者未分配空间，该空间就是等会安装ubuntu的位置。

2.制作U盘启动盘：

在此步骤之前，确保安装的系统镜像已经下载好了，去ubuntu中文官方网站即可下载最新的版本18.04，我目前用的就是这个版本。然后还需要一个不小于4G的U盘。
制作启动盘的软件我选用的是powerISO，去官网下载，安装选择试用。
软件打开后，菜单栏点击“打开”，选择你下载好的系统镜像，然后选择制作U盘启动盘，写入模式建议选择USB-HDD，点击“写入”，一般需要15min，看到刻录完成即成功。

3.重启系统进入BIOS，选择U盘启动：

每种型号的计算机进入BIOS的方式一般不一样，thinkpad的笔记本启动后看到屏幕下方有提示，按Ehter键进入选择菜单，然后可以看到最后一行英文意思是选择一种临时启动方式，正是我们所需要的，这样就不用依照网上有些教程所说的设置启动优先级，安装完了还要修改回来，我们只需要临时一次就可以安装好。选择了U盘启动后，就会启动U盘里的系统，等一会儿，出现一个界面，让你选择是试用ubuntu还是安装ubuntu，选择安装。

4.安装方式的选择：

按照提示设置，安装时提示你联网，选择不联网，选择不要更新软件，这样可以节省很多时间。其中有一个设置界面是选择安装方式，第一个选择是与windows共存，第二个和第三个会覆盖wildows系统–意思是所有磁盘数据会清空，给你安装一个崭新的ubuntu，还有最后一个选项是其他，网上教程一般让你选择其他，如果选择其他，则需要自己分区，提前要想好根目录、/home、/boot、swap分区的大小，如果选择第一种，则会自行分区，并且不需要选择安装位置，它自行寻找一个空的盘符，就是我们在第一步中已经预留好的空间，/root即根目录会和你预留的安装空间差不多大，/home在根目录下.

记于九月初

发表于 2017-09-03 | 分类于 随笔

永远不要高估自己的自律能力

这段时间情绪低落，主要原因是没能完成自己暑期任务，这让我再度开始怀疑自己的能力，一直以来都很钦佩那些给自己定下目标后就能按时完成的人，就像写程序一样，本来是不可控的东西，但总有人能突破极限，在极短的时间写出效率极高的代码，然而这样的人凤毛菱角，显然我不属于这类人。

我回顾自己的有意识的人生，发现自己有几个极大的缺陷，做事效率不高，经常自视过高，这可能是梦想家的通病吧，只不过我病的更深一点。不久前，曾与身边一位“做事效率极高”的朋友谈到这个，他说这可能是习惯问题，起初我不太赞同，我觉得这可能和智商一样，很难改变，反思自己的暑期备考经历，我开始对他的这种说法感到赞同。

同样的在自习室，同样的一个小时，有的人可以读完2篇千字以上的外刊，有的人却一篇都读不完，抛开对单词语法的熟练程度不算，这很大程度上是由习惯决定的，有些人习惯不停地思考问题，而有些人思想总开小差，我对那些能三四个小时注意力高度集中在一件事情上的人感到震惊，因为我可能半小时都做不到，当我发现自己像个多动症儿童一样，我感到羞愧，我知道习惯很难改变，但越是困难的事，完成后的成就感越大。

很多事情我明白了，但我不喜欢说于他人听，因为实际的行动比其他任何事情都重要，人生一途，“过程”是自己的，“结果”是他人的，关于考研，过程、结果，我都要。与诸君共勉！

has查找思想

发表于 2017-08-21 | 分类于 算法思想

散列（hash)是一种重要的算法思想，hash函数在计算机领域应用非常广泛，下面主要谈谈其在字符查找方面的一些应用。

当我们需要在一个域里面查找一些元素时，最简单的思想是遍历整个域，将查询元素与域内元素进行判断是否一致，这项工作相当耗时，我们不妨这样想，将查询元素与查询域建立一个联系，构建一个函数关系hash(X),X为待查元素，hash(X)=0(初始化)为没有X元素,hash(X)=1为X元素出现一次，这样我们可以直接得出任意元素n是否出现过(hash(n)是否为0)，但这也出现一个问题，就是我们需要一个非常大的数组来表示这个函数，这也称为“以空间换时间”。

因此，当查询域很大时，这个方法并不适用。

那么怎么进行字符串查找呢？

当我们给出两个字符串，需要查找后者在前者中出现过，依据上面介绍的思想，我们创建一个数组hashTable[max],接下来，我们需要把字符串转换为数字，作为数组的下标，转换后可能出现的情况的总和即为查找域的大小max,因此max可能很大，占据了非常多的空间，但绝大多数是无用的。
